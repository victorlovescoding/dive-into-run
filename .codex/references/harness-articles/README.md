## Source-Fidelity Protocol

When rewriting or refreshing article summaries:

1. Read the source text before editing. Do not rewrite from memory or from another agent's paraphrase.
2. Verify semantic-critical structure against the source: arrows (`->`, `<->`, `→`, `↔`), dependency direction, layer order, graph edges, table rows, and named categories.
3. If the source has a diagram or table, preserve its structure or explicitly say the summary is simplified.
4. Mark uncertain or historical items as provenance, not current rule. Use phrasing like "source says" or "historical note" when it is not an active repo rule.
5. After editing, search for critical symbols and terms in both source and summary. A changed arrow direction or dependency edge is a correctness bug, not a style issue.

Reference summaries are transcription-heavy work. Source fidelity beats smoother prose.
