# Feature Specification: Run Calendar

**Feature Branch**: `008-run-calendar`
**Created**: 2026-04-07
**Status**: Draft
**Input**: User description: "在 runs 頁面建立月曆按鈕，點擊後顯示當月月曆，以三種不同 SVG 圖示標示戶外跑步、室內跑步、越野跑的紀錄及公里數，底部顯示當月總里程"

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 查看當月跑步月曆 (Priority: P1)

身為已連接 Strava 的使用者，我想在 runs 頁面點擊月曆按鈕後看到當月的月曆視圖，快速掌握這個月哪幾天有跑步、每天跑了多少公里。

**Why this priority**: 月曆視圖是本功能的核心價值，讓使用者一目了然地檢視整月跑步分布。

**Independent Test**: 登入並連接 Strava 後，進入 runs 頁面，點擊月曆按鈕，確認彈出月曆 dialog 並顯示當月日曆格子，有跑步紀錄的日期顯示對應圖示與公里數。

**Acceptance Scenarios**:

1. **Given** 使用者已登入且已連接 Strava，**When** 進入 runs 頁面，**Then** 頁面 header 區域顯示月曆按鈕。
2. **Given** 使用者點擊月曆按鈕，**When** 月曆 dialog 開啟，**Then** 顯示當前月份的日曆，標題顯示「YYYY年M月」格式。
3. **Given** 當月某天有跑步紀錄，**When** 查看該日期格子，**Then** 格子以淡綠底色標示，並顯示活動類型 SVG 圖示 + 公里數（如跑者圖示 + 6.0）。
4. **Given** 當月某天無任何跑步紀錄，**When** 查看該日期格子，**Then** 格子僅顯示日期數字，無圖示。

---

### User Story 2 - 區分三種跑步類型 (Priority: P1)

身為使用者，我想在月曆中透過不同圖示區分戶外跑步、室內跑步、越野跑三種活動類型，了解我的訓練多樣性。

**Why this priority**: 三種跑步類型的視覺區分是使用者明確要求的核心需求。

**Independent Test**: 在有包含不同跑步類型紀錄的月份中，確認月曆格子以不同圖示正確標示各類型。

**Acceptance Scenarios**:

1. **Given** 某天有戶外跑步紀錄，**When** 查看該日期格子，**Then** 顯示跑者 SVG 圖示 + 公里數。
2. **Given** 某天有室內跑步紀錄，**When** 查看該日期格子，**Then** 顯示跑步機 SVG 圖示 + 公里數。
3. **Given** 某天有越野跑紀錄，**When** 查看該日期格子，**Then** 顯示人在山林跑步 SVG 圖示 + 公里數。
4. **Given** 同一天有多種跑步類型，**When** 查看該日期格子，**Then** 各類型分行顯示各自的圖示 + 公里數。
5. **Given** 同一天有同種類型的多筆紀錄，**When** 查看該日期格子，**Then** 該類型的公里數為加總值。

---

### User Story 3 - 查看當月跑步總結 (Priority: P1)

身為使用者，我想在月曆底部看到當月的跑步里程總結，快速知道這個月總共跑了多少公里。

**Why this priority**: 月度總結提供直觀的成就感回饋，是月曆功能的重要配套。

**Independent Test**: 開啟月曆後，確認底部顯示當月所有跑步（三種類型合計）的總里程。

**Acceptance Scenarios**:

1. **Given** 當月有跑步紀錄，**When** 查看月曆底部，**Then** 顯示跑者 SVG 圖示 + 當月總里程（三種類型合計），格式如「跑者圖示 14.3 km」，以及各類型小計。
2. **Given** 當月無任何跑步紀錄，**When** 查看月曆底部，**Then** 顯示「跑者圖示 0.0 km」或相應的零里程提示。
3. **Given** 當月僅有部分類型的跑步紀錄，**When** 查看月曆底部，**Then** 僅顯示有紀錄之類型的小計，無紀錄的類型不顯示。

---

### User Story 4 - 切換月份 (Priority: P2)

身為使用者，我想在月曆中切換到上一個月或下一個月，回顧不同月份的跑步紀錄。

**Why this priority**: 月份切換是月曆的標準功能，但核心價值在前三個 Story 的當月展示。

**Independent Test**: 開啟月曆後，點擊上一個月/下一個月按鈕，確認月曆更新為對應月份的資料。

**Acceptance Scenarios**:

1. **Given** 月曆顯示 2026 年 4 月，**When** 點擊上一個月按鈕，**Then** 月曆更新為 2026 年 3 月的日曆及跑步紀錄。
2. **Given** 月曆顯示 2026 年 1 月，**When** 點擊上一個月按鈕，**Then** 月曆更新為 2025 年 12 月（跨年處理正確）。
3. **Given** 切換到另一個月份，**When** 資料載入中，**Then** 顯示載入指示器。
4. **Given** 切換月份後，**When** 資料載入完成，**Then** 月份標題、日曆格子、底部總結同步更新為該月份的資料。

---

### Edge Cases

- 使用者未登入或未連接 Strava → 不顯示月曆按鈕
- 同一天有 3 種類型各多筆紀錄 → 各類型公里數各自加總，分 3 行顯示
- 某月份完全沒有任何活動 → 空白日曆 + 底部顯示 0.0 km
- 切換月份時網路錯誤 → 顯示錯誤提示，保留上一個月份的顯示
- 跨年月份切換（12月 ↔ 1月） → 年份正確更新

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 系統必須在 runs 頁面 header 區域顯示月曆按鈕，僅在使用者已連接 Strava 時可見
- **FR-002**: 點擊月曆按鈕時，開啟月曆 dialog，預設顯示當前月份。Desktop 為置中 modal，mobile 為全螢幕（100vw × 100vh）
- **FR-003**: 月曆以 7 欄網格呈現（日、一、二、三、四、五、六），正確對齊該月第一天的星期
- **FR-004**: 有跑步紀錄的日期格子須以淡綠底色標示，並顯示活動類型圖示 + 公里數
- **FR-005**: 三種跑步活動類型須以不同 SVG 圖示區分：戶外跑步（跑者）、室內跑步（跑步機）、越野跑（人在山林中跑步）
- **FR-006**: 同一天有多種跑步類型時，各類型在格子內分行顯示各自的圖示 + 公里數
- **FR-007**: 同一天同種類型有多筆紀錄時，該類型的公里數顯示為加總值
- **FR-008**: 月曆底部顯示當月跑步總里程（三種類型公里數合計）附跑者 SVG 圖示，並顯示各類型小計（如「戶外 8.1 / 室內 3.0 / 越野 3.2」）
- **FR-009**: 提供上一個月 / 下一個月切換按鈕，切換後月份標題、日曆、底部總結同步更新
- **FR-010**: 月份資料載入過程中顯示載入指示器
- **FR-011**: 資料載入失敗時顯示錯誤提示
- **FR-012**: 月曆資料透過 Firestore 月份範圍 query 取得（`uid` + `startDate` >= 月初 & < 下月初），不依賴現有分頁查詢
- **FR-013**: 日期格為純展示，不可點擊，無互動行為
- **FR-014**: 三種活動類型 SVG 圖示為自製 inline SVG component，不引入外部 icon library

### Key Entities

- **跑步活動 (Running Activity)**: 使用者透過 Strava 同步的跑步紀錄。關鍵屬性：活動日期、距離（公里）、活動類型（戶外跑步 / 室內跑步 / 越野跑）
- **月曆日期格 (Calendar Day Cell)**: 月曆中代表一天的格子。關鍵屬性：日期、是否有跑步、各類型的公里數

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 使用者能透過月曆按鈕開啟月曆 dialog，一眼看到當月的跑步分布
- **SC-002**: 三種跑步類型可透過不同圖示一眼區分，無需額外說明
- **SC-003**: 月曆資料載入在 2 秒內完成
- **SC-004**: 月份切換後資料在 2 秒內更新
- **SC-005**: 月曆底部的總里程與各類型小計數值正確反映該月所有跑步紀錄

## Assumptions

- 月曆僅顯示當前登入使用者自己的跑步資料
- 公里數顯示至小數點後一位（如 6.0、14.3）
- 月曆 dialog 可透過點擊背景或關閉按鈕關閉
- 月份切換無範圍限制（可往前往後任意切換）
- 不提供周視圖或年視圖，僅月視圖

- Q: 日曆日期格點擊行為？ → A: 純展示，日期格不可點擊
- Q: Dialog 在 mobile 的呈現方式？ → A: Desktop 置中 modal，mobile 全螢幕（100vw × 100vh）
- Q: 月曆底部月摘要是否顯示各類型小計？ → A: 總里程 + 各類型小計（僅顯示有紀錄的類型）
- Q: SVG 圖示來源？ → A: 自製 inline SVG component，不引入外部 icon library
