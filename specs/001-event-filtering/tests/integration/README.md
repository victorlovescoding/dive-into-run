# 整合測試：活動篩選介面 (Integration Testing)

**目標範圍**: 活動列表頁面、篩選元件、地圖交互。
**測試重點**: 元件間的協作與使用者操作行為。

## 規範 (Kent C. Dodds Style)
- **必須使用**: **Testing Library 三劍客** (`dom`, `react`, `user-event`)。
- **禁止使用**: `fireEvent`。
- **哲學**: 模擬使用者真實操作（如：點擊按鈕、在搜尋框打字）。

## 預計測試情境
1. **篩選連動**: 使用者在篩選選單選擇縣市後，列表應自動更新。
2. **地圖同步**: 使用者縮放地圖後，列表是否根據範圍變動（若有此功能）。
3. **錯誤處理**: 當 Firebase 連線失敗時，畫面上是否出現正確的錯誤提示。
