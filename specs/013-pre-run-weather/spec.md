## 跑步前天氣頁面 (Pre-Run Weather Page)

**Feature Branch**: `013-pre-run-weather`
**Created**: 2026-04-11
**Status**: Draft
**Input**: User description: "根據 pre-run-weather-prd.md 建立站內輕量天氣頁面,讓使用者用地圖選擇跑步地點並查看今日/明日天氣,並可收藏常用地點"

## User Scenarios & Testing _(mandatory)_

### User Story 1 — 選地點查天氣 (Priority: P1)

身為一個習慣在固定河濱或市區跑步的跑者,我在出門前想快速查一下今天要去的地區天氣如何,決定穿什麼、要不要帶毛巾、會不會被曬爆。我希望打開這個頁面就看到一張完整的台灣地圖——包含本島、金門、馬祖、澎湖以及蘭嶼、綠島等離島——點一下我要去的地區,旁邊就顯示該地區今天和明天的完整天氣資訊,不用跳到氣象局網站或別的 app。如果我要去蘭嶼這種小離島,我點了蘭嶼就要直接看到蘭嶼的天氣,不需要先知道蘭嶼屬於哪個縣市再一層一層找。

**Why this priority**: 這是整個頁面存在的根本目的。地圖必須在第一個畫面就完整呈現台灣全貌(含離島),否則視覺上殘缺、對離島使用者不友善。離島縣市(金門/連江/澎湖)本身就是獨立行政區,與本島縣市同等待遇;小離島(蘭嶼/綠島/小琉球/龜山島)則提供一鍵直達天氣的智能路由,避免使用者需要知道行政歸屬才能查天氣。

**Independent Test**: 進入天氣頁面,地圖呈現台灣全台總覽(含本島 22 縣市、金門/連江/澎湖 3 離島縣市、蘭嶼/綠島/小琉球/龜山島 4 小離島標示)。點擊本島縣市(例如臺北市),驗證天氣卡顯示該縣市天氣。點擊離島縣市(例如金門縣),驗證同等行為。點擊小離島標示(例如蘭嶼),驗證地圖切到臺東縣鄉鎮層、蘭嶼鄉自動選中浮起、天氣卡直接顯示蘭嶼鄉天氣。整個流程不需要登入。

**Acceptance Scenarios**:

1. **Given** 使用者首次進入天氣頁面, **When** 頁面載入完成, **Then** 地圖呈現全台總覽,涵蓋本島 22 縣市、3 離島縣市(連江縣、金門縣、澎湖縣)及 4 小離島標示(蘭嶼、綠島、小琉球、龜山島),共 29 個可互動單位,天氣卡顯示「請先在地圖上選擇想查詢的地區」的空狀態
2. **Given** 使用者在全台總覽地圖上, **When** 游標 hover 到某個縣市或離島標示, **Then** 該區塊底色由預設淺天空藍 (`#EAF8FC`) 溫柔漸變為較深天空藍 (`#A4C3E4`),其他區塊維持原色
3. **Given** 使用者在全台總覽地圖上, **When** 點擊臺北市, **Then** 天氣卡顯示該縣市整體的今日完整天氣資訊(當前氣溫、天氣狀況、早晚氣溫、降雨、濕度、UV、AQI)與明日摘要(早晚氣溫、降雨、濕度、UV)
4. **Given** 使用者在全台總覽地圖上, **When** 點擊金門縣(或連江縣、澎湖縣), **Then** 天氣卡顯示該離島縣市整體的今日完整天氣資訊與明日摘要,行為與點擊本島縣市完全相同
5. **Given** 使用者在全台總覽地圖上, **When** 點擊蘭嶼標示, **Then** 地圖切換到臺東縣鄉鎮地圖,蘭嶼鄉以「已選中」狀態呈現(浮起),天氣卡直接顯示蘭嶼鄉的完整天氣,地點名稱為「臺東縣 · 蘭嶼鄉」
6. **Given** 使用者在全台總覽地圖上, **When** 點擊綠島標示, **Then** 地圖切換到臺東縣鄉鎮地圖,綠島鄉已選中浮起,天氣卡直接顯示綠島鄉天氣
7. **Given** 使用者在全台總覽地圖上, **When** 點擊小琉球標示, **Then** 地圖切換到屏東縣鄉鎮地圖,琉球鄉已選中浮起,天氣卡直接顯示琉球鄉天氣
8. **Given** 使用者在全台總覽地圖上, **When** 點擊龜山島標示, **Then** 地圖切換到宜蘭縣鄉鎮地圖,頭城鎮已選中浮起,天氣卡直接顯示頭城鎮天氣,地點名稱明確標示為「宜蘭縣 · 頭城鎮(含龜山島)」
9. **Given** 天氣卡正在載入資料, **When** 資料尚未回傳, **Then** 卡片顯示與實際尺寸一致的骨架佔位,避免版面跳動
10. **Given** 天氣資料載入失敗, **When** 錯誤發生, **Then** 天氣卡內顯示「無法取得天氣,請稍後再試」提示與重試按鈕,不用 toast
11. **Given** 使用者點擊某地區後天氣載入中, **When** 使用者立刻點擊另一個地區, **Then** 前一次請求的結果被捨棄,最終只顯示後點擊地區的天氣

### User Story 2 — 下鑽鄉鎮看更精確的天氣 (Priority: P2)

身為一個特別在意跑步地點天氣的跑者,我光看縣市整體天氣不夠,因為新北市板橋區和烏來區的天氣差很多。我希望點了縣市之後可以再點進去看某個鄉鎮的天氣,並且隨時可以回到全台總覽重新選。

**Why this priority**: 縣市整體天氣已能滿足大多數場景(P1),但在縣市範圍大或山海差異明顯的地區(新北、臺北、花蓮、臺東等),鄉鎮級的天氣能顯著提升資訊價值。這層互動讓頁面從「夠用」變成「貼身」。可獨立測試:不依賴收藏功能。

**Independent Test**: 從全台總覽點擊某縣市(例如新北市),驗證地圖切換到該縣市的鄉鎮地圖,左上角出現「全台總覽」回退按鈕;點擊某個鄉鎮後該鄉鎮視覺浮起,天氣卡更新為該鄉鎮天氣;點擊「全台總覽」按鈕可回到全台地圖並清空狀態。

**Acceptance Scenarios**:

1. **Given** 使用者在全台總覽地圖上, **When** 點擊新北市, **Then** 地圖切換到新北市鄉鎮地圖,顯示該縣市所有鄉鎮區;左上角出現「全台總覽」按鈕(含台灣輪廓 icon 與文字);天氣卡顯示新北市整體天氣
2. **Given** 使用者在新北市鄉鎮地圖上, **When** 游標 hover 到某個鄉鎮, **Then** 該鄉鎮底色由 `#EAF8FC` 溫柔漸變為 `#A4C3E4`
3. **Given** 使用者在新北市鄉鎮地圖上, **When** 點擊板橋區, **Then** 板橋區視覺浮起(帶陰影與微微位移),天氣卡更新為板橋區的完整天氣
4. **Given** 板橋區已被選中且浮起, **When** 使用者點擊烏來區, **Then** 板橋區回到平面,烏來區浮起,天氣卡更新為烏來區天氣
5. **Given** 板橋區已被選中, **When** 使用者再次點擊板橋區, **Then** 不做任何事(不取消選中、不重新載入)
6. **Given** 使用者在新北市鄉鎮地圖上, **When** 點擊左上角「全台總覽」按鈕, **Then** 地圖切回全台總覽層,已選縣市和鄉鎮狀態被清空,天氣卡回到空狀態

### User Story 3 — 收藏常查的地點並快速切換 (Priority: P3)

身為一個會固定在幾個地方跑步的跑者,我每次都要從全台地圖上找那幾個點很麻煩。我希望能收藏常查的縣市或鄉鎮,而且頁面上有一個收藏區塊,一眼看到我收藏的所有地點以及它們當前的天氣概況(天氣圖示 + 氣溫),點一下就能切到該地點的完整天氣。不想要的收藏也能在區塊裡直接移除。

**Why this priority**: 這是提升留存和回訪效率的加值功能。收藏區塊讓使用者不用每次從全台地圖重新找,已收藏的地點一鍵切換。未登入使用者要能看到收藏按鈕的存在以利功能發現,但收藏區塊只在有收藏項時才顯示。桌面版收藏區塊縱向排列在天氣卡上方;手機版以橫向可捲動列表放在地圖上方,節省垂直空間。

**Independent Test**: 登入後選中多個地點(例如臺北市、新北市板橋區、臺東縣蘭嶼鄉)並逐一收藏,驗證收藏區塊出現且每項顯示地點名稱 + 天氣圖示 + 氣溫 + ✕ 取消按鈕;點擊收藏項(非 ✕)驗證地圖切換到該地點並更新天氣卡;點擊 ✕ 驗證該項從區塊中移除並彈出取消收藏 toast;未登入狀態下驗證收藏按鈕可見但點擊只彈 toast 不動作。

**Acceptance Scenarios**:

1. **Given** 使用者未登入且天氣卡已顯示某地點天氣, **When** 點擊收藏按鈕, **Then** 彈出 toast「請先登入才能收藏」,不執行收藏動作,不強制跳轉登入頁
2. **Given** 已登入使用者選中新北市板橋區且該地點尚未收藏, **When** 點擊天氣卡右上角收藏按鈕, **Then** 按鈕 icon 由空心變為實心並伴隨填滿動畫,彈出 toast「已收藏」,收藏區塊同步新增該地點項目
3. **Given** 板橋區已收藏, **When** 再次點擊天氣卡上的收藏按鈕, **Then** 按鈕 icon 變回空心,彈出 toast「取消收藏」,收藏區塊同步移除該地點項目
4. **Given** 已登入使用者有多個收藏(例如臺北市、板橋區、蘭嶼鄉), **When** 進入天氣頁面, **Then** 收藏區塊出現,每個項目顯示地點名稱、天氣狀況圖示、當前氣溫與 ✕ 取消按鈕
5. **Given** 收藏區塊已顯示多個收藏項, **When** 使用者點擊「臺東縣 · 蘭嶼鄉」收藏項(非 ✕ 按鈕), **Then** 地圖切換到臺東縣鄉鎮層並預選蘭嶼鄉,天氣卡更新為蘭嶼鄉的完整天氣
6. **Given** 收藏區塊已顯示多個收藏項, **When** 使用者點擊某項的 ✕ 按鈕, **Then** 該項從收藏區塊中移除,彈出 toast「取消收藏」,天氣卡上的收藏按鈕狀態同步更新(若當前查看的就是被取消的地點)
7. **Given** 已登入使用者同時收藏新北市整體與新北市板橋區, **When** 查看收藏區塊, **Then** 兩者為獨立的收藏項,同時出現在區塊中
8. **Given** 已登入使用者剛完成收藏動作, **When** 重新整理頁面, **Then** 收藏按鈕和收藏區塊的狀態均正確還原
9. **Given** 使用者點擊收藏或取消收藏時後端寫入失敗, **When** 錯誤發生, **Then** UI 先樂觀更新,失敗後回滾狀態(含收藏區塊),並彈出 toast「操作失敗,請稍後再試」
10. **Given** 已登入使用者沒有任何收藏, **When** 進入天氣頁面, **Then** 收藏區塊不顯示(不佔空間)
11. **Given** 使用者未登入, **When** 進入天氣頁面, **Then** 收藏區塊不顯示(不佔空間)

### Edge Cases

- **龜山島的歸屬歧義**:龜山島在行政劃分上屬於宜蘭縣頭城鎮,不是獨立鄉鎮。UI 必須明確顯示「宜蘭縣 · 頭城鎮(含龜山島)」讓使用者知道被對應到哪,避免「我點龜山島為什麼跑出頭城鎮」的困惑。
- **縣市無鄉鎮資料的保險處理**:理論上每個縣市都有鄉鎮區劃,但若資料異常(例如 API 回傳空陣列),退回顯示該縣市整體天氣,不進入鄉鎮層。
- **連續快速切換地區**:載入中又切換時,舊請求的結果必須被捨棄,只保留最新選擇的資料,避免畫面閃爍或錯誤更新。
- **未登入使用者看到收藏按鈕**:天氣卡上的收藏按鈕必須顯示而非隱藏,讓使用者感知此功能存在,點擊時才提示登入。收藏區塊本身不顯示(因為未登入不可能有收藏資料)。
- **收藏區塊天氣資料載入失敗**:若進頁面時部分收藏項的天氣資料拉取失敗,該項仍顯示地點名稱與 ✕ 按鈕,氣溫和天氣圖示顯示佔位「—」,可點擊切換仍可用。
- **收藏數量很多**:桌面版縱向列表超出可視高度時可捲動;手機版橫向滾動天生無上限。第一版不設收藏數量上限。
- **從收藏區塊刪除當前正在查看的地點**:天氣卡仍保持顯示該地點天氣(不清空),但天氣卡上的收藏按鈕同步變為未收藏。
- **桌面版與手機版版面差異**:桌面版天氣卡在左、地圖在右並排;手機版地圖在上、天氣卡在下。手機版點擊鄉鎮後頁面自動 smooth scroll 到天氣卡位置,方便使用者看到結果。
- **手機版地點名稱長度**:手機版地點名稱縮寫(例:「臺東 · 蘭嶼」而非「臺東縣 · 蘭嶼鄉」),避免版面過擠。龜山島情境下仍須保留「含龜山島」的標示。
- **空氣品質為何只在今日、不在明日**:預報資料的 AQI 準確度不足,列出反而誤導。明日區塊刻意不顯示 AQI。
- **刻意不提供跑步適合度評分**:PRD 明確排除「適合 / 不適合跑步」的綜合評分,因為定義主觀且雞肋。本頁只提供原始天氣數據,由使用者自行判斷。

## Requirements _(mandatory)_

### Functional Requirements

#### 地圖層(全台總覽)

- **FR-001**: 頁面 MUST 預設顯示台灣全台總覽地圖,涵蓋本島 22 個縣市及 3 個離島縣市(連江縣、金門縣、澎湖縣),共 25 個縣市(依縣市邊界切割),每個縣市為可互動的獨立單位
- **FR-002**: 全台總覽地圖 MUST 另含 4 個小離島可點擊單位:蘭嶼、綠島、小琉球、龜山島,各自位於地理實際座標位置,視覺上 MUST 清晰可辨識且易於點擊
- **FR-003**: 全台總覽地圖所有區塊的預設底色 MUST 為 `#EAF8FC`(淺天空藍),無任何選中狀態
- **FR-004**: 使用者 hover 任一縣市或離島標示時,該區塊底色 MUST 溫柔漸變為 `#A4C3E4`(深天空藍),其他區塊維持原色,顏色變化不得瞬間切換
- **FR-005**: 使用者點擊任一縣市(本島 22 縣市及連江、金門、澎湖)時,地圖 MUST 切換到該縣市的鄉鎮地圖層
- **FR-006**: 使用者點擊蘭嶼標示時,地圖 MUST 切換到臺東縣鄉鎮地圖,蘭嶼鄉 MUST 自動選中(視覺浮起),天氣卡 MUST 同時載入並顯示蘭嶼鄉天氣
- **FR-007**: 使用者點擊綠島標示時,地圖 MUST 切換到臺東縣鄉鎮地圖,綠島鄉 MUST 自動選中(視覺浮起),天氣卡 MUST 同時載入並顯示綠島鄉天氣
- **FR-008**: 使用者點擊小琉球標示時,地圖 MUST 切換到屏東縣鄉鎮地圖,琉球鄉 MUST 自動選中(視覺浮起),天氣卡 MUST 同時載入並顯示琉球鄉天氣
- **FR-009**: 使用者點擊龜山島標示時,地圖 MUST 切換到宜蘭縣鄉鎮地圖,頭城鎮 MUST 自動選中(視覺浮起),天氣卡 MUST 同時載入並顯示頭城鎮天氣,且地點名稱 MUST 明確顯示為「宜蘭縣 · 頭城鎮(含龜山島)」

#### 地圖層(縣市鄉鎮)

- **FR-010**: 縣市鄉鎮地圖 MUST 顯示該縣市所有鄉鎮區,依鄉鎮邊界切割,每個鄉鎮為可互動的獨立單位
- **FR-011**: 縣市鄉鎮地圖上鄉鎮底色規則 MUST 與全台總覽相同:預設 `#EAF8FC`,hover 變為 `#A4C3E4`
- **FR-012**: 進入縣市鄉鎮地圖時,若使用者是從小離島標示(蘭嶼/綠島/小琉球/龜山島)進來,對應鄉鎮 MUST 以「已選中」狀態呈現(視覺浮起),天氣卡 MUST 同時自動載入並顯示該鄉鎮天氣,使用者不需再手動點選
- **FR-013**: 使用者點擊某鄉鎮時,該鄉鎮 MUST 視覺上浮起(帶陰影與微微位移),底色 MUST 維持 `#A4C3E4`,天氣卡同時更新為該鄉鎮天氣
- **FR-014**: 使用者點擊另一個鄉鎮時,前一個鄉鎮 MUST 回到平面,新鄉鎮浮起
- **FR-015**: 使用者點擊已選中的鄉鎮時,系統 MUST NOT 取消選中(不做 toggle,不重新載入)
- **FR-016**: 使用者在縣市鄉鎮地圖上但尚未選擇任一鄉鎮時,天氣卡 MUST 顯示該縣市整體天氣,不得呈現空白

#### 「全台總覽」回退按鈕

- **FR-017**: 縣市鄉鎮地圖左上角 MUST 顯示「全台總覽」按鈕,按鈕內容包含台灣輪廓 icon(上層)與文字「全台總覽」(下層)
- **FR-018**: 「全台總覽」按鈕 MUST 只在縣市鄉鎮層顯示,全台總覽層不得顯示
- **FR-019**: 使用者點擊「全台總覽」按鈕時,地圖 MUST 切回全台總覽層,已選縣市與已選鄉鎮狀態 MUST 被清空,天氣卡 MUST 回到初始空狀態

#### 天氣資訊卡

- **FR-020**: 天氣卡未選任何地區時 MUST 顯示空狀態:含視覺插畫與文字「請先在地圖上選擇想查詢的地區」,風格與整個頁面一致(不得為純文字)
- **FR-021**: 天氣卡資料載入中 MUST 顯示與實際卡片尺寸一致的骨架佔位狀態,帶 shimmer 動畫,避免版面跳動 (layout shift)
- **FR-022**: 天氣卡資料載入失敗 MUST 在卡片內顯示錯誤提示「無法取得天氣,請稍後再試」並提供重試按鈕,不得使用 toast
- **FR-023**: 今日天氣區塊 MUST 顯示以下欄位:地點名稱、當前氣溫(視覺最突出)、天氣狀況文字、早上氣溫、晚上氣溫、降雨機率、相對濕度、空氣品質 (AQI,含數字與等級文字)、紫外線指數 (UV,含數字與等級文字)
- **FR-024**: 明日天氣區塊 MUST 顯示以下欄位:早上氣溫、晚上氣溫、降雨機率、相對濕度、紫外線指數(含等級文字)
- **FR-025**: 明日天氣區塊 MUST NOT 顯示空氣品質 (AQI),因為預報資料準確度不足
- **FR-026**: 天氣卡視覺層級 MUST 依以下順序由最突出到最淡:(1) 當前氣溫 + 天氣狀況 (2) 今日早晚氣溫 (3) 今日四個指標(降雨、濕度、UV、AQI) (4) 明日摘要
- **FR-027**: 當使用者載入中又切換地區時,系統 MUST 取消前一次請求的結果,只顯示最新選擇地區的資料

#### 收藏按鈕(天氣卡上)

- **FR-028**: 收藏按鈕 MUST 位於天氣卡右上角
- **FR-029**: 收藏按鈕 MUST 有兩種狀態:未收藏(空心 outline 書籤 icon)與已收藏(實心 filled 書籤 icon)
- **FR-030**: 未登入使用者點擊收藏按鈕時,系統 MUST 彈出 toast「請先登入才能收藏」,不執行收藏動作,不強制導向登入頁
- **FR-031**: 未登入使用者 MUST 仍能看到收藏按鈕(不得隱藏),以利功能發現
- **FR-032**: 已登入使用者由未收藏轉為收藏時,按鈕 icon MUST 填滿並彈出 toast「已收藏」,收藏區塊同步新增該地點項目
- **FR-033**: 已登入使用者由已收藏轉為未收藏時,按鈕 icon MUST 變回空心並彈出 toast「取消收藏」,收藏區塊同步移除該地點項目
- **FR-034**: 使用者 MUST 能獨立收藏縣市(鄉鎮未選狀態)與單一鄉鎮,兩者為獨立收藏項;同一縣市與其下鄉鎮可同時被收藏
- **FR-035**: 收藏動作 MUST 採樂觀更新策略:UI 先反映新狀態(含天氣卡按鈕與收藏區塊),後端寫入失敗時回滾並彈出 toast「操作失敗,請稍後再試」
- **FR-036**: 重新整理頁面或再次造訪已收藏地點時,收藏按鈕與收藏區塊狀態 MUST 正確還原

#### 收藏區塊(天氣頁內嵌)

- **FR-049**: 天氣頁面 MUST 包含一個收藏區塊,位於天氣卡所在面板內;桌面版位於天氣卡上方(縱向列表),手機版位於地圖上方(橫向可捲動列表)
- **FR-050**: 收藏區塊 MUST 只在已登入使用者有至少一個收藏時才顯示;未登入使用者或無收藏時不佔空間
- **FR-051**: 收藏區塊每個項目 MUST 顯示:地點名稱、天氣狀況圖示、當前氣溫、取消收藏按鈕 (✕)
- **FR-052**: 進入頁面時,收藏區塊 MUST 載入並顯示所有收藏地點的當前天氣圖示與氣溫;載入中時可顯示佔位狀態
- **FR-053**: 使用者點擊收藏項(✕ 以外的區域)MUST 觸發地圖切換到該地點並更新天氣卡:若為縣市級收藏,切到該縣市鄉鎮層並顯示縣市整體天氣;若為鄉鎮級收藏,切到該鄉鎮所屬縣市鄉鎮層並預選該鄉鎮
- **FR-054**: 使用者點擊收藏項的 ✕ 按鈕 MUST 取消該地點的收藏,該項從收藏區塊中移除,彈出 toast「取消收藏」;若當前天氣卡顯示的正是被取消的地點,天氣卡上的收藏按鈕 MUST 同步更新為未收藏狀態
- **FR-055**: 手機版收藏區塊以橫向可捲動列表呈現,每項為緊湊的 chip 形態,✕ 按鈕的觸控目標 MUST 至少為 44×44px
- **FR-056**: 桌面版收藏區塊以縱向列表呈現,收藏項超過可視區域高度時 MUST 可捲動
- **FR-057**: 收藏區塊的視覺風格 MUST 與整體頁面 Soft Sky 風格一致(淺天空藍底色、圓潤、airy)
- **FR-058**: 收藏區塊中的收藏項 MUST 按收藏時間倒序排列(最新收藏的排最前面)

#### 響應式與跨裝置

- **FR-037**: 桌面版 (螢幕寬度 ≥ 1024px) 版面 MUST 為天氣卡在左、地圖在右並排
- **FR-038**: 手機版 (螢幕寬度 < 768px) 版面 MUST 為地圖在上、天氣卡在下
- **FR-039**: 平板版 (中等尺寸) MUST 採用與手機版相同的上下排列佈局,不另做獨立版面
- **FR-040**: 桌面版地點名稱 MUST 完整顯示(例:「臺東縣 · 蘭嶼鄉」)
- **FR-041**: 手機版地點名稱 MUST 縮寫(例:「臺東 · 蘭嶼」,去除縣/市/鄉/鎮後綴),但龜山島情境須保留「(含龜山島)」標示
- **FR-042**: 手機版使用者點擊鄉鎮或點擊收藏區塊中的收藏項後,頁面 MUST 自動 smooth scroll 到天氣卡位置
- **FR-043**: 手機版收藏按鈕的觸控目標 MUST 至少為 44×44px

#### 視覺設計

- **FR-044**: 整體視覺 MUST 遵循 Soft Sky 風格:天空藍主導、留白大於資料密度、圓潤大於銳利、陰影使用淡天空藍而非黑色
- **FR-045**: 地圖與卡片的預設底色 MUST 使用 `#EAF8FC`,hover 與選中狀態 MUST 使用 `#A4C3E4`
- **FR-046**: 大氣溫數字與地點名稱 MUST 使用襯線體或 display 字型(例如 Fraunces、Playfair Display、Instrument Serif 類型),與 app 現有其他頁面形成視覺差異
- **FR-047**: 資料標籤、小字、數字(百分比、指數) MUST 使用無襯線體或等寬體,與 app 現有頁面一致
- **FR-048**: 所有動畫 MUST 溫柔、不誇張,不得出現彈跳、旋轉、翻轉、視差滾動、3D 變形等效果(鄉鎮浮起可有輕微彈性)

#### URL 狀態同步

- **FR-059**: 頁面 URL MUST 為 `/weather`,使用者選擇縣市或鄉鎮時,系統 MUST 透過 `history.replaceState` 將選中狀態同步到 URL query params(例如 `/weather?county=taipei&township=xinyi`),不觸發 Next.js 頁面重新渲染
- **FR-060**: 使用者直接以含 query params 的 URL 進入頁面時,系統 MUST 從 query params 解析初始狀態,自動導航到對應的縣市/鄉鎮並載入天氣
- **FR-061**: 使用者點擊「全台總覽」回退時,URL query params MUST 被清空,回到 `/weather`
- **FR-062**: URL 狀態同步 MUST 使用 `replaceState`(不建立新 browser history entry),瀏覽器返回鍵 MUST 離開天氣頁面回到上一頁,不回到全台總覽層;使用者透過「全台總覽」按鈕回上層

#### 頁面初始地點還原

- **FR-063**: 系統 MUST 在使用者每次查看某地點天氣時,將該地點(縣市及鄉鎮,若有)儲存至瀏覽器 localStorage,此行為適用於所有使用者(不論登入與否、有無收藏)
- **FR-064**: 已登入且有收藏的使用者進入 `/weather` 且 URL 無 query params 時,系統 MUST 將 localStorage 上次查看地點與收藏清單比對:(a) 若上次查看地點存在於收藏清單中,自動還原該收藏地點;(b) 若上次查看地點不在收藏清單中或 localStorage 無紀錄,自動顯示收藏清單中的第一個項目(按 FR-058 排序,即最新收藏的地點)
- **FR-064a**: 未登入使用者或已登入但無收藏的使用者進入 `/weather` 且 URL 無 query params 時,系統 MUST 從 localStorage 讀取上次查看的地點並自動還原;若 localStorage 無紀錄(首次造訪或已清除),MUST 顯示空狀態(FR-020)
- **FR-065**: 自動還原為鄉鎮級地點時,地圖 MUST 切換到該鄉鎮所屬縣市的鄉鎮層,且該鄉鎮以已選中狀態呈現(視覺浮起);還原為縣市級地點時,地圖 MUST 切換到該縣市的鄉鎮層並顯示縣市整體天氣
- **FR-066**: 頁面初始地點的優先順序 MUST 為:(1) URL query params(FR-060)> (2) 已登入有收藏:localStorage 上次地點若命中收藏則顯示該收藏,否則顯示第一個收藏(FR-064)> (3) 未登入或無收藏:localStorage 上次地點(FR-064a)> (4) 空狀態(FR-020)

### Key Entities

- **地圖地點 (Map Location)**:代表使用者可在地圖上點選的地理單位,分三層:(1) 本島 22 縣市 + 3 離島縣市(連江/金門/澎湖)+ 4 離島標示(蘭嶼/綠島/小琉球/龜山島);(2) 縣市底下的鄉鎮區;(3) 離島標示與其行政歸屬的鄉鎮對應關係(如龜山島 → 宜蘭縣頭城鎮)。每個地點都有唯一識別、顯示名稱(桌面/手機縮寫版)、層級、以及對應的天氣查詢鍵。
- **天氣資訊 (Weather Info)**:某地點在特定時間點的天氣快照,包含今日完整欄位(當前氣溫、天氣狀況文字、早晚氣溫、降雨機率、濕度、UV、AQI)與明日摘要(早晚氣溫、降雨、濕度、UV,無 AQI)。
- **收藏地點 (Favorite Location)**:一位已登入使用者對某個地圖地點的收藏紀錄。縣市層級與其下鄉鎮為獨立項,可共存。持久化至使用者帳號,跨裝置同步。收藏區塊中每項附帶該地點的即時天氣摘要(天氣狀況圖示 + 當前氣溫),進頁面時批次載入。

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 使用者從打開頁面到看到所選地區的天氣,完成時間 MUST 不超過 **2 次點擊**(全台地圖 → 點縣市 → 看到該縣市天氣),鄉鎮級查詢不超過 3 次點擊;小離島查詢(蘭嶼/綠島/小琉球/龜山島) MUST 不超過 **1 次點擊**(全台地圖 → 點小離島標示 → 直接看到該鄉鎮天氣)
- **SC-002**: 所選地區的天氣卡 MUST 在 **2 秒內**完成資料載入並呈現(正常網路條件下)
- **SC-003**: 全部 4 個離島智能路由情境(蘭嶼、綠島、小琉球、龜山島) MUST 100% 正確對應到隸屬縣市與預選鄉鎮,無錯誤路由
- **SC-004**: 未登入使用者點擊收藏按鈕時,系統 MUST 不會導向登入頁,僅以 toast 提示,確保瀏覽流程不被打斷
- **SC-005**: 已登入使用者的收藏狀態在重新整理頁面或跨裝置登入後 MUST 100% 正確還原
- **SC-006**: 頁面在 **桌面版 (≥1024px)**、**平板版 (768-1023px)**、**手機版 (<768px)** 三種尺寸下 MUST 均可正常操作,不出現版面錯亂或觸控目標過小等問題
- **SC-007**: 使用者在完成一次查詢後, MUST 能在 1 次點擊內(「全台總覽」按鈕)回到起點重新選擇其他地區
- **SC-009**: 已有收藏的使用者打開天氣頁面, MUST 能在 **1 次點擊**內(點擊收藏項)看到任一常用地點的完整天氣,不需要重新在地圖上找
- **SC-008**: 視覺風格 MUST 通過 Soft Sky 主觀驗收(天空藍主導、無暖色污染、大氣溫數字使用 display serif 字型),形成與 app 現有頁面的視覺差異

## Assumptions

- **天氣資料來源**:假設有可取得台灣縣市與鄉鎮級今日/明日天氣資料的穩定資料源(例如中央氣象署開放資料 API 或第三方天氣服務),確切來源與實作方式留到 plan 階段決定
- **縣市整體天氣的定義**:當使用者選了縣市但未選鄉鎮時,該縣市的「整體天氣」採用氣象局提供的縣市層級預報值(通常是縣市代表點),不額外做鄉鎮資料的平均
- **今日時段定義**:「早上氣溫」與「晚上氣溫」採用氣象局預報資料原生的時段劃分(例如白天 / 夜間),不額外自訂時段
- **頁面瀏覽權限**:未登入訪客可自由瀏覽地圖並查看所有地區天氣,登入僅為收藏功能所需
- **收藏持久化**:收藏資料儲存於使用者帳號,跨裝置同步;未登入使用者的收藏意圖不保留
- **收藏上限**:第一版不設收藏數量上限
- **定位服務**:第一版不自動偵測使用者當前位置,由使用者主動在地圖上點選
- **資料更新策略**:每次進入頁面或切換地區時,向資料源抓取最新天氣;不做長時間快取。使用者停留在頁面期間不自動更新,需手動重新整理頁面取得最新資料
- **行政區資料**:台灣本島 22 縣市與鄉鎮行政區劃資料(邊界 GeoJSON 或類似格式)假設可從公開來源取得
- **離島標示的視覺定位**:蘭嶼、綠島、小琉球、龜山島在全台總覽地圖上的顯示位置,依地理實際座標繪製標記,不要求與縣市底圖完全等比例
